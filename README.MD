## git이란?

git은 분산 버전 관리 시스템(DVCS)으로 소스 코드의 변경 이력을 추적하고 관리하는 도구이다.

git과 같은 프로그램으로 svn 등 여러 서비스가 존재하지만 여기서는 git만을 사용할 것이다.

## git을 왜 사용하는가?

분명 어떠한 기술이 새로생기고 추가가 되었다면 그 이전 환경 또는 기술에서 문제점이 있거나 필요성을 느꼈을 것인데 과연 어떠한 문제점이 있었을까
그 3가지를 아래서 하나씩 살펴보자.

### 1. 버전 관리의 문제

사진 : p1.webp 

```
프로젝트 완료.zip
프로젝트 추가 완료.zip
프로젝트 진짜 완료.zip
프로젝트 찐찐 완료 10월10일.zip
프로젝트 로그인 수정 10월15일.zip
프로젝트 버그수정 진짜 완료 10월20일.zip
프로젝트 롤백 후 수정 10월25일.zip
```

이렇게 제출을 위해 또는 프로젝트 공유를 위해 관리를 해본적이 있는가?

처음엔 **완료**를 적었다가 그 다음 **진짜완료** **진짜진짜완료**가 붙고 이후로는 단어로 안되니깐 **날짜**가 추가되고 그마저 부족하니깐 뭘 수정했는지 **로그**를 추가하고 하다보니 결국은 **롤백**을 해서 수정을 하는 사진이다.

이렇게 우리가 git을 쓰지 않으면 이런 버전 관리에서 문제가 생길수 있다.

---

### 2. 동시 작업의 문제

사진 : p2.png

```
팀원 : 나 A 파일 수정했어
나 : 어 나도 그 파일 수정했는데
팀원 : 그럼 수정한 코드 전달해줄래?
나 : 여기 있어 (A파일 전체)
팀원 : (복붙 후 수정...)
```

협업을 위해 친구와 작업을 하는데 각자 수정을 하다보니 같은 파일을 수정하게 될 경우 이런 상황이 생기게 된다.

카카오톡으로 길게 붙여넣은 코드를 복사 붙여넣기 해서 작업하는데 그렇게 잘 실행되면 그나마 다행이지 간혹가다가 이마저도 실행이 안된다

알고보니 자바, 파이썬, 자바스크립트의 경우 import c, c++ 과 같은 경우는 include가 빠져서 이런 문제가 생기기도 하고 추가된 코드가 나와 같은 코드거나 한 함수 안에 서로 다르게 수정하다보니 뭔가 이상하게 꼬이는경우도 생길 수 있다.

---

### 3. 이력 추적의 문제

사진 : p3.png

```
아... 언제 이 코드를 넣었더라?
이 코드는 왜 이렇게 된거지?

```

본인이 작성한 코드를 1주일 뒤에 보게되면 정말 이게 내가 작성한건가 싶을 정도로 기억이 안나는 경우가 많다.

혼자해도 이런데 팀원과 같이 하게된다면 어떤 코드를 언제 추가했고 왜 추가했고 하는 이유를 주석으로 하나하나 남기지 않으면 정말 기억하기가 쉽지않다.

---

이런 문제들이 있기에 git을 사용하기에 배워야 하는 것이며 모든 기업에서는 기본 소양으로 git 사용 능력을 요구하니 여러분들은 취업을 위해서라도 배워야 하는 기술이다.


## 배우게 될 키워드

이번 시간에서 여러분들이 배우게 되는 키워드는 다음과 같다
📦 1. 저장소 설정

init - 새 저장소 만들기
clone - 기존 저장소 복사
remote - 원격 저장소 연결
.gitignore - 추적 제외 설정

📝 2. 기본 작업 흐름

status - 현재 상태 확인
add - 변경사항 준비
commit - 변경사항 저장
push - 원격에 업로드
pull - 원격에서 다운로드
fetch - 정보만 가져오기

🌿 3. 브랜치 & 병합

branch - 작업 공간 만들기
checkout - 작업 공간 이동
merge - 작업 합치기
stash - 임시 저장

🔍 4. 협업 & 디버깅

log - 커밋 히스토리
diff - 변경사항 비교
blame - 수정자 추적
Pull Request(PR) - 코드 리뷰
conflict - 충돌 해결

익혀야하는 용어나 단어가 많아 복잡하겠지만 이것 하나하나 다 따라할 수 있도록 만들어주는데다가 일부 명령어는 깃헙 데스크톱을 사용해서 대체하게 될것이니 크게 걱정하지 않아도 된다.

## 혼자 하는 git

깃에서 관리하는 하나의 프로젝트 단위를 **레파지토리** 라고 한다.

먼저 레파지토리를 생성을 하면 
사진 : solo1.png
와 같이 나타나며

1. 레파지토리 이름을 작성
2. 설명은 선택적으로 작성
3. 프로젝트의 공개여부에 따라 public 또는 private 선택
4. 나머지는 추후 본인들이 깨닫게 되면 옵션을 조절하면 되고
5. create repository 버튼을 클릭하면 레파지토리가 생성된다.
